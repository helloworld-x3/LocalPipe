# 调研综述

Hello World 团队 · 易点天下「AI创意本地化流水线」命题

## 1. 行业现状

- 纯人工本地化单条素材耗时 11-18 天，已被市场淘汰；当前主流 AI 本地化流水线（翻译、配音、字幕、合规一条龙）已将单条成本压至百元级（腾讯云媒体AI公开计费方案）
- 但现有流水线只做**形式转换**：同一条创意擦水印、换语言、换配音铺多国，本质是一稿多投，传统流程文化适配准确率不足 65%
- 超五成出海企业因本地化不足丢失过业务；87% 的企业确认本地化投资直接推动扩张成功（Phrase《2025全球扩张指南》）
- 行业成本模型已完成「纯人工 → 纯AI → AI+人工审校」迭代，AI+人工混合模式降本 72% 以上，是 90% 出海团队的当前最优解——本方案在此共识上再进一步：从「形式转换+人工审校」升级为「创意重创作+结构化验证」

## 2. 竞品扫描

| 产品/服务 | 能力 | 空位 |
|-----------|------|------|
| 腾讯云媒体AI（MAIS） | 擦除/翻译/配音/字幕/扩画面/审核全流水线 | 创意本身不变，形式转换 |
| 易营宝 | 22语种素材制作，1200+敏感词库 | 翻译思维，无创意重做 |
| MotionPoint Transcreation（2025.9发布） | AI创译+母语者审核 | 面向欧美市场，无中国创意出海视角 |
| Rask.ai 文化适配框架 | 视频本地化+文化指标分析 | 视频形式层，非创意层 |

**结论**：市场上「快」和「多」已经有答案，「接地气」（创意层重创作）是明确空位。

## 3. 开源同类项目扫描（GitHub）

报名前我们检索了 GitHub 上 localization / transcreation / i18n-agent 相关仓库，确认差异性：

- **Brand-Loom**：营销文案生成器，带 brand_context 设计（品牌调性约束）——本项目的 `brand_context.json` 借鉴了这一模式，并叠加了保护术语锁定
- **Tower of Babel**：翻译流水线，带术语表锁定（glossary enforcement）——本项目的 `protected_terms` 逐项回检借鉴了这一模式
- 未发现任何仓库具备：保真闭环回检（产出反向解构 vs 源要素比对+自动打回）、画像条目元数据化（置信度/时效/来源/ID）、产出级画像引用追溯（used_entries）——这三项是本项目的差异化机制

## 4. 学术依据

1. **Kocaman, "Lost in Transcreation"（Kenyon College, 2025）**：GPT-5.2 / Claude 4.5 / Gemini 3 / Grok-4 四个前沿模型做跨文化广告创译，100 分制无一超过 75 分，全部在文化共鸣维度失败——证明裸 LLM 无法解决「接地气」，需要结构化文化上下文与验证机制
2. **Unbabel, MT Summit 2025**：GPT-4o 广告转译对照实验，20 条中带文化上下文 Prompt 在 17 条上优于裸 Prompt——直接支撑本方案「文化画像注入」的技术路径
3. **EC Innovations 中英本地化基准（2026）**：774 条产出、五种工作流对比，结论：原始 LLM 输出不足以达到企业级标准，行业正从「工具中心」转向「工作流中心」；营销内容上中文模型（Qwen/豆包）优于西方模型——本方案路线图中的模型路由设计据此选型
4. **Hassani et al., ELOPE 2025**：翻译学生经六周针对性 AI 训练后创译质量反超职业翻译——人机协作方法论的实证支撑
5. **MPTc-Bench（ACL 2026 Findings）**：34 市场电影海报视觉转译基准，模型产出与本地真实海报存在实质性差距——视觉本地化作为规划扩展方向，已有学术基准可依

## 5. 结论

裸模型搞不定文化共鸣（学术依据 1），结构化文化上下文有效（学术依据 2），工作流比单模型重要（学术依据 3）。三条证据共同指向本方案的技术路径：**画像库底座 + 分层管线 + 闭环验证**。
